# generate-insight-summary

## 목적

이 스킬은 계산된 지표와 해석된 투자 신호를 바탕으로 사용자가 바로 이해할 수 있는 설명 문장을 생성하는 역할을 한다.

이 문서는 특정 프로그램의 문구 명세가 아니라, 투자 데이터 분석 결과를 안전하고 일관되게 설명하기 위한 범용 요약 기준서이다.

이 스킬의 목적은 다음과 같다.

- 계산된 숫자와 상태 판정을 사람이 이해하기 쉬운 문장으로 변환한다.
- 대시보드에 표시할 핵심 요약 문장을 생성한다.
- 상태의 근거를 간단하고 명확하게 설명한다.
- 위험 신호가 있을 경우 주의 문구를 함께 생성한다.
- 투자 초보자도 이해할 수 있도록 쉬운 설명을 별도로 생성한다.
- 시각화 또는 문서 출력에서 사용할 수 있도록 표준 설명 구조를 반환한다.

이 스킬은 **설명 생성 역할**을 한다.  
이 스킬은 수치를 계산하지 않는다.  
이 스킬은 상태를 새로 판정하지 않는다.  
이 스킬은 매수/매도와 같은 투자 추천을 하지 않는다.

---

## 사용할 때

다음 상황에서 이 스킬을 사용한다.

- `detect-market-status`가 완료된 뒤
- 대시보드 상단 요약 문구가 필요할 때
- 현재 시장 상태를 한 줄로 설명해야 할 때
- 상태 판정의 근거를 사용자에게 보여줘야 할 때
- 초보자용 쉬운 설명이 필요할 때
- 위험 신호를 경고 문구로 보여줘야 할 때

즉, 이 스킬은 **계산 이후, 상태 판정 이후, 최종 출력 직전 단계**에서 사용한다.

---

## 사용하지 않아도 되는 경우

다음 조건을 모두 만족하면 생략할 수 있다.

- 설명 문장을 사람이 수동으로 직접 작성한다.
- 대시보드가 숫자와 상태 라벨만 보여주면 충분하다.
- 사용자용 자연어 설명이 필요 없다.

단, 본 프로젝트는 초보자 친화적 대시보드이므로 이 스킬을 사용하는 것을 권장한다.

## 입력 데이터

이 스킬은 기본적으로 아래 두 결과를 함께 사용한다.

1. `calculate-core-metrics`의 계산 결과
2. `detect-market-status`의 상태 판정 결과

권장 입력 구조는 다음과 같다.

```json
{
  "metrics_result": {
    "status": "success",
    "metrics": {
      "current_close": 152.3,
      "daily_return": 1.24,
      "weekly_return": 3.82,
      "monthly_return": null,
      "cumulative_return": 8.15,
      "ma5": 149.8,
      "ma20": 145.2,
      "ma60": null,
      "volatility_20d": 12.4,
      "volume_change_rate": -8.2,
      "price_vs_ma20_pct": 4.89,
      "ma5_vs_ma20_pct": 3.17
    },
    "meta": {
      "record_count": 30,
      "latest_date": "2026-05-03"
    }
  },
  "status_result": {
    "status": "success",
    "market_status": "up",
    "trend_strength": "medium",
    "volatility_level": "medium",
    "risk_signal": false,
    "risk_reasons": [],
    "reason_codes": [
      "PRICE_ABOVE_MA20",
      "MA5_ABOVE_MA20",
      "WEEKLY_RETURN_POSITIVE"
    ],
    "reason_text": "Current price is above the 20-day moving average, short-term average is also above the mid-term average, and weekly return is positive."
  }
}
```

## 필수 입력 조건

아래 조건을 만족해야 한다.

- `metrics_result` 존재
- `status_result` 존재
- `status_result.market_status` 존재
- `status_result.status`가 `success` 또는 `partial_success`
- 최소한 일부 핵심 수치가 존재
  - `daily_return`
  - `weekly_return`
  - `ma20`
  - `volatility_20d`
  - `price_vs_ma20_pct`
  - `ma5_vs_ma20_pct`

## 이 스킬이 생성해야 하는 출력

이 스킬은 아래 항목을 생성해야 한다.

1. `headline`
2. `summary`
3. `insight`
4. `caution`
5. `beginner_note`
6. `status_label`
7. `confidence_note`
8. `explanation_points`

설명:

- `headline`: 가장 짧은 핵심 한 줄
- `summary`: 현재 상태에 대한 핵심 설명
- `insight`: 상태가 의미하는 바를 한 단계 더 설명
- `caution`: 주의가 필요한 경우 경고 문장, 주의가 없으면 `null`
- `beginner_note`: 초보자용 쉬운 설명
- `status_label`: UI 표시용 짧은 상태 문구
- `confidence_note`: 해석 강도나 데이터 충분성 설명
- `explanation_points`: UI 카드/툴팁용 짧은 근거 리스트

## 핵심 작성 원칙

### 1. 상태 먼저, 이유는 그 다음

문장은 항상 아래 순서를 우선한다.

1. 현재 상태 제시
2. 왜 그렇게 보았는지 설명
3. 주의할 점이 있으면 보완 설명

좋은 예:
- "현재 흐름은 상승입니다. 최근 가격이 20일 평균 위에 있고 주간 수익률도 양수입니다. 다만 변동성이 높다면 단기 해석에는 주의가 필요합니다."

나쁜 예:
- "20일 이동평균과 주간 수익률과 변동성 등을 종합적으로 고려할 때..."
- 상태가 늦게 나오는 문장은 피한다.

---

### 2. 숫자와 해석을 함께 연결

숫자를 단순 나열하지 말고 의미와 함께 표현한다.

좋은 예:
- "현재 가격은 20일 평균보다 약 4.9% 높은 수준입니다."

나쁜 예:
- "price_vs_ma20_pct = 4.89"

원칙:
- 내부 값은 숫자로 유지
- 사용자용 문장은 해석된 자연어로 변환

---

### 3. 과도한 단정 금지

다음 표현은 사용하지 않는다.

- "반드시 오른다"
- "곧 하락한다"
- "지금 사야 한다"
- "매도해야 한다"

대신 다음 표현을 사용한다.

- "상승 흐름으로 해석됩니다"
- "주의가 필요한 구간입니다"
- "방향성이 비교적 약합니다"
- "단기 해석에는 신중할 필요가 있습니다"

---

### 4. 초보자 친화적 설명 우선

전문 용어를 그대로 반복하지 않는다.

예:
- "변동성이 높다"만 쓰지 말고
- "가격이 많이 흔들리고 있다는 뜻입니다"를 함께 제공

---

### 5. 상태 라벨과 설명 문장을 분리

- `status_label`은 짧고 단순해야 한다.
- `summary`, `insight`, `beginner_note`는 문장 형태로 제공한다.

예:
- `status_label = 상승`
- `summary = 현재 가격 흐름은 전반적으로 상승 방향으로 해석됩니다.`

---

### 6. 근거 코드 기반으로 작성

요약 문장은 `reason_codes`와 `risk_reasons`를 기준으로 작성한다.

정확 기준:

- `reason_codes`가 있으면 explanation point는 reason code 순서대로 생성한다.
- 짧은 요약에는 근거를 최대 3개만 포함한다.
- 상세 설명에는 근거를 최대 5개만 포함한다.
- `risk_reasons`가 1개 이상이면 `caution`을 반드시 채운다.
- 없는 수치나 `null` 지표는 문장에 언급하지 않는다.

---

### 7. 공통 투자 신호 라벨

가격 시계열이 아닌 투자 데이터에는 아래 공통 라벨을 사용한다.

| signal | status_label | headline |
|---|---|---|
| `positive` | 긍정 | 긍정 신호 |
| `negative` | 부정 | 부정 신호 |
| `neutral` | 중립 | 중립 구간 |
| `caution` | 주의 | 주의 필요 |
| `insufficient_data` | 데이터 부족 | 데이터 부족 |
| `invalid_data` | 데이터 오류 | 분석 불가 |

가격 시계열 데이터는 `up`, `down`, `sideways`를 각각 상승, 하락, 횡보로 표현할 수 있다.

## 상태별 문장 생성 규칙

## 1. 상승 상태 (`market_status = up`)

### headline
- "상승 흐름"
- "완만한 상승 흐름"
- "상승 우세"

### summary 생성 원칙
다음 요소를 포함한다.

- 현재 상태가 상승이라는 점
- 현재 가격이 평균선 위에 있다는 점
- 주간 수익률이 양수라는 점

예시:
- "현재 가격 흐름은 전반적으로 상승 방향으로 해석됩니다."
- "최근 가격은 20일 평균 위에서 움직이며 상승 흐름을 보이고 있습니다."

### insight 생성 원칙
다음 요소를 포함할 수 있다.

- 단기 평균이 중기 평균 위에 있다는 점
- 최근 성과가 양수라는 점
- 상승 흐름의 강도

예시:
- "단기 평균도 중기 평균보다 높아 최근 흐름이 비교적 강한 편입니다."
- "주간 수익률이 양수여서 최근 가격 흐름에 긍정적인 신호가 있습니다."

### caution 생성 원칙
상승 상태라도 아래 경우 caution을 보완한다.

- `volatility_level = high`
- `risk_signal = true`

예시:
- "다만 가격 변동성이 높다면 단기 해석에는 주의가 필요합니다."

### beginner_note 예시
- "상승 흐름이라는 것은 최근 가격이 전반적으로 위쪽을 향하고 있다는 뜻입니다."

---

## 2. 하락 상태 (`market_status = down`)

### headline
- "하락 흐름"
- "약세 흐름"
- "하락 우세"

### summary 생성 원칙
다음 요소를 포함한다.

- 현재 상태가 하락이라는 점
- 현재 가격이 평균선 아래라는 점
- 최근 수익률이 음수라는 점

예시:
- "현재 가격 흐름은 전반적으로 하락 방향으로 해석됩니다."
- "최근 가격은 20일 평균 아래에서 움직이며 약세 흐름을 보이고 있습니다."

### insight 생성 원칙
다음 요소를 포함할 수 있다.

- 단기 평균도 중기 평균보다 낮다는 점
- 주간 기준 성과가 약하다는 점

예시:
- "단기 평균도 중기 평균 아래에 있어 최근 흐름은 비교적 약한 편입니다."
- "주간 수익률이 음수여서 최근 가격 압력이 아래쪽에 있는 상태입니다."

### caution 생성 원칙
하락 상태에서 `volatility_level = high`이면 변동성 주의 문장을 제공한다.

예시:
- "최근 변동성도 큰 편이라 가격 흐름이 더 불안정하게 나타날 수 있습니다."

### beginner_note 예시
- "하락 흐름이라는 것은 최근 가격이 전반적으로 아래쪽을 향하고 있다는 뜻입니다."

---

## 3. 횡보 상태 (`market_status = sideways`)

### headline
- "횡보 구간"
- "방향성 약함"
- "중립 구간"

### summary 생성 원칙
다음 요소를 포함한다.

- 뚜렷한 방향성이 없다는 점
- 가격이 평균 부근에서 움직인다는 점

예시:
- "현재 가격 흐름은 뚜렷한 상승이나 하락 없이 횡보 구간으로 해석됩니다."
- "가격이 평균 부근에서 움직이고 있어 방향성이 비교적 약한 상태입니다."

### insight 생성 원칙
다음 요소를 포함할 수 있다.

- 단기 평균과 중기 평균 차이가 작음
- 주간 수익률 절대값이 작음
- 관망 구간 가능성

예시:
- "단기 평균과 중기 평균의 차이가 크지 않아 시장 방향성이 뚜렷하지 않습니다."
- "최근 성과 변화도 크지 않아 관망 구간으로 볼 수 있습니다."

### caution 생성 원칙
변동성이 낮으면 안정적 중립 설명, 높으면 불안정 횡보 설명을 제공한다.

예시:
- "방향성은 약하지만 변동성이 높다면 짧은 기간 내 흔들림은 클 수 있습니다."

### beginner_note 예시
- "횡보 구간은 가격이 크게 오르거나 내리지 않고 비슷한 범위에서 움직이는 상태입니다."

---

## 4. 주의 상태 (`market_status = caution`)

### headline
- "주의 필요"
- "변동성 주의"
- "불안정 구간"

### summary 생성 원칙
다음 요소를 포함한다.

- 지금은 주의가 필요한 상태라는 점
- 위험 신호의 핵심 원인 1~2개
- 과장 없이 설명

예시:
- "최근 가격 변동성이 커 주의가 필요한 구간입니다."
- "최근 낙폭과 변동성이 함께 커져 불안정성이 높아진 상태입니다."

### insight 생성 원칙
다음 요소를 포함할 수 있다.

- 급락 여부
- 고변동성 여부
- 평균선 대비 큰 이탈 여부
- 거래량 급증 여부

예시:
- "현재 가격이 평균선 아래로 크게 이탈했거나 최근 하락 폭이 커 단기 리스크가 높아질 수 있습니다."
- "거래량이 급증한 가운데 가격 변화도 커져 시장 반응이 예민한 상태로 볼 수 있습니다."

### caution 생성 원칙
`market_status = caution`이면 `caution` 필드는 거의 항상 채운다.

예시:
- "이 구간은 방향성보다 변동 위험을 먼저 확인하는 것이 좋습니다."
- "짧은 기간에 가격이 크게 흔들릴 수 있어 해석에 신중할 필요가 있습니다."

### beginner_note 예시
- "주의 상태는 가격이 갑자기 크게 움직이거나 불안정해서 조심해서 봐야 한다는 뜻입니다."

---

## 5. 데이터 부족 상태 (`market_status = insufficient_data`)

### headline
- "데이터 부족"
- "분석 정보 부족"

### summary 예시
- "현재 데이터만으로는 시장 상태를 안정적으로 해석하기 어렵습니다."

### insight 예시
- "이동평균이나 변동성처럼 핵심 판단에 필요한 정보가 충분하지 않습니다."

### caution 예시
- "데이터가 더 쌓인 뒤 다시 해석하는 것이 적절합니다."

### beginner_note 예시
- "지금은 판단에 필요한 정보가 부족해서 결과를 단정하기 어렵다는 뜻입니다."

### headline

### summary 예시

### caution 예시
- "원본 데이터를 다시 확인한 뒤 재분석해야 합니다."

### beginner_note 예시
- "지금은 데이터 자체에 문제가 있어서 결과를 신뢰하기 어렵습니다."

## 문장 길이 규칙

### headline
- 2~12자 내외의 짧은 문구
- UI 카드 제목으로 바로 쓸 수 있어야 함

### summary
- 1문장
- 35~90자 내외 권장

### caution
- 필요할 때만 생성
- 1문장
- 30~100자 내외 권장

### beginner_note
- 1문장
- 쉬운 말 위주
- 전문 용어 반복 최소화

## 숫자 표현 규칙

이 스킬은 숫자를 사용자 친화적으로 표현해야 한다.

### 퍼센트 수치
- 기본적으로 소수점 1자리 또는 2자리까지 허용
- 소수점 2자리를 초과한 표시 금지

예:
- `4.8921` → `4.9%`
- `-2.034` → `-2.0%`

### 가격
- 필요한 경우 현재 가격을 포함할 수 있다.
- 단, 문장마다 가격 숫자를 반복하지 않는다.

### 이동평균 비교
- `price_vs_ma20_pct`는 아래처럼 자연어로 바꾼다.
  - 양수 → "20일 평균보다 약 X% 높은 수준"
  - 음수 → "20일 평균보다 약 X% 낮은 수준"

### 변동성
- 내부 수치 자체보다 `low / medium / high` 해석값을 우선 사용한다.
- 숫자가 꼭 필요한 경우만 함께 언급한다.

## 출력 필드 설명

### `status`

스킬 실행 상태

가능한 값:

- `success`
- `partial_success`
- `failed`

원칙:

- 문장 생성이 정상 완료되면 `success`
- 일부 정보만으로 제한 생성 시 `partial_success`
- 입력 오류 등으로 생성 불가 시 `failed`

---

### `status_label`

UI 표시용 짧은 상태 라벨

가능한 값 예:

- `상승`
- `하락`
- `횡보`
- `주의`
- `데이터 부족`
- `데이터 오류`

원칙:

- 짧고 명확하게 유지
- `market_status`의 한국어 표현으로 사용

---

### `headline`

대시보드 카드 제목 또는 요약 상단 문구

원칙:

- 매우 짧게 작성
- 현재 상태가 한눈에 보이도록 작성

---

### `summary`

현재 상태에 대한 핵심 한 줄 설명

원칙:

- 최종 상태를 가장 먼저 드러내야 함
- 너무 긴 복합문 피하기
- 사용자 메인 화면용 문장

---

### `insight`

상태의 근거를 한 단계 더 설명하는 문장

원칙:

- 계산값의 의미를 자연어로 변환
- summary보다 조금 더 구체적이어야 함
- 1~2문장 허용

---

### `caution`

주의 문구

원칙:

- 주의할 점이 없으면 `null`
- `market_status = caution`이면 반드시 채움
- 상승/하락/횡보 상태에서도 보조 위험이 있으면 채울 수 있음

---

### `beginner_note`

초보자용 쉬운 설명

원칙:

- 전문 용어를 쉬운 말로 바꿈
- 설명 난이도를 낮춤
- 교육용/툴팁용으로도 활용 가능

---

### `confidence_note`

해석 신뢰도 설명

예시:

- "핵심 지표가 비교적 충분해 현재 해석의 신뢰도는 보통 이상입니다."
- "핵심 지표가 일부 부족해 현재 해석은 제한적으로 참고하는 것이 좋습니다."

원칙:

- 데이터 충분성 또는 근거 강도를 짧게 설명

---

### `explanation_points`

짧은 근거 포인트 목록

예시:

- "현재 가격이 20일 평균 위에 있습니다."
- "주간 수익률이 양수입니다."
- "변동성은 보통 수준입니다."

원칙:

- 2~4개 권장
- 각 항목은 짧고 직관적으로 작성
- 툴팁, 리스트, 설명 카드에 바로 사용할 수 있어야 함

---

### `used_metrics`

문장 생성에 실제 사용한 핵심 수치 목록

목적:

- 어떤 값이 문장 생성에 반영되었는지 명시
- 디버깅과 검증 용도

---

### `source_status`

상태 판정 결과 원본 요약

포함 권장 항목:

- `market_status`
- `trend_strength`
- `volatility_level`
- `risk_signal`

---

### `meta`

부가 메타 정보

포함 권장 항목:

- `latest_date`
- `record_count`

## 문장 생성 절차

이 스킬은 아래 순서로 동작한다.

1. 입력 구조가 유효한지 검사한다.
2. `market_status` 값을 확인한다.
3. 상태별 기본 템플릿을 선택한다.
4. 사용 가능한 핵심 수치를 확인한다.
5. `summary`를 생성한다.
6. `insight`를 생성한다.
7. 위험 신호 여부를 보고 `caution`을 생성한다.
8. 초보자용 쉬운 설명을 생성한다.
9. 근거 포인트를 2~4개 추출한다.
10. 최종 출력 구조로 정리한다.

## 상태별 템플릿 선택 규칙

템플릿 후보가 여러 개 있으면 첫 번째로 조건이 맞는 템플릿을 사용한다. 같은 입력에서 임의로 다른 후보를 선택하지 않는다.

### `market_status = up`
사용 우선 템플릿:
- 상승 상태 템플릿
- 상승 + 보통 변동성 템플릿
- 상승 + 고변동성 보완 템플릿

### `market_status = down`
사용 우선 템플릿:
- 하락 상태 템플릿
- 하락 + 고변동성 보완 템플릿

### `market_status = sideways`
사용 우선 템플릿:
- 횡보 상태 템플릿
- 횡보 + 안정 템플릿
- 횡보 + 불안정 보완 템플릿

### `market_status = caution`
사용 우선 템플릿:
- 주의 상태 템플릿
- 급락 주의 템플릿
- 고변동성 주의 템플릿
- 거래량 급증 주의 템플릿

### `market_status = insufficient_data`
사용 우선 템플릿:
- 데이터 부족 템플릿

### `market_status = invalid_data`
사용 우선 템플릿:
- 데이터 오류 템플릿

## 설명 포인트 생성 규칙

`explanation_points`는 아래 우선순위로 생성한다.

### 상승일 때 우선 포인트
- 가격이 20일 평균 위
- 5일 평균이 20일 평균 위
- 주간 수익률 양수
- 변동성 보통 이하

### 하락일 때 우선 포인트
- 가격이 20일 평균 아래
- 5일 평균이 20일 평균 아래
- 주간 수익률 음수
- 최근 하락 압력 존재

### 횡보일 때 우선 포인트
- 가격이 평균 부근
- 단기/중기 평균 차이가 작음
- 주간 변화 폭이 크지 않음

### 주의일 때 우선 포인트
- 변동성 높음
- 최근 급락
- 평균선 대비 큰 이탈
- 거래량 급증 + 가격 급변

---

## confidence_note 생성 규칙

### 높은 신뢰도
조건 예:
- `record_count >= 20`
- `ma20` 있음
- `weekly_return` 있음
- `volatility_20d` 있음

예시:
- "핵심 지표가 충분해 현재 해석의 신뢰도는 비교적 높습니다."

### 보통 신뢰도
조건 예:
- `ma20`, `weekly_return`, `volatility_20d` 중 2개 이상 있음
- `reason_codes`가 2개 이상 있음

예시:
- "핵심 지표가 대체로 확보되어 현재 해석은 보통 수준으로 참고할 수 있습니다."

### 낮은 신뢰도
조건 예:
- `ma20`, `weekly_return`, `volatility_20d` 중 2개 이상 없음
- `reason_codes`가 1개 이하임

예시:
- "일부 지표가 부족해 현재 해석은 제한적으로 참고하는 것이 좋습니다."

## 예외 처리 규칙

### 1. 입력 구조 오류

조건:
- `metrics_result` 없음
- `status_result` 없음
- `market_status` 없음

처리:
- `status = failed`
- `headline = 데이터 오류`
- `summary = 입력 구조가 올바르지 않아 설명을 생성할 수 없습니다.`

---

### 2. 상태는 있으나 수치가 거의 없는 경우

조건:
- `market_status`는 존재
- `ma20`, `weekly_return`, `volatility_20d` 중 2개 이상 없음

처리:
- 상태 중심의 짧은 설명만 생성
- `insight`는 최소화
- `confidence_note`에서 해석 제한을 명시

---

### 3. 주의 상태이지만 원인이 여러 개인 경우

조건:
- 고변동성 + 급락 + 거래량 급증 등이 함께 존재

처리:
- 가장 중요한 원인 1~2개만 summary/insight에 반영
- 나머지는 `explanation_points` 또는 `caution`에 반영
- 문장을 과도하게 길게 만들지 않는다

---

### 4. 월간 수익률 등 일부 값이 없는 경우

처리:
- 없는 값을 억지로 언급하지 않는다.
- 사용 가능한 지표만으로 문장 생성
- 문장 자연스러움을 우선한다.

---

## 금지 사항

이 스킬은 다음을 수행하지 않는다.

- 수치 계산
- 상태 재판정
- 임계값 재정의
- 매수 / 매도 / 보유 추천
- 종목 선택 추천
- 과도한 확정 표현 사용
- 뉴스/외부 이벤트 추론

즉, 이 스킬은 **설명 문장 생성 전용 스킬**이다.

## 예시 시나리오

### 1. 상승 상태 예시

입력 상태:
- `market_status = up`
- `price_vs_ma20_pct = 4.89`
- `ma5_vs_ma20_pct = 3.17`
- `weekly_return = 3.82`
- `volatility_level = medium`

출력 예시:
- `headline`: 상승 흐름
- `summary`: 현재 가격 흐름은 전반적으로 상승 방향으로 해석됩니다.
- `insight`: 현재 가격은 20일 평균보다 높은 수준에 있고, 단기 평균도 중기 평균 위에 있어 최근 흐름이 비교적 강한 편입니다.
- `caution`: ""
- `beginner_note`: 상승 흐름은 최근 가격이 전반적으로 위쪽을 향하고 있다는 뜻입니다.

---

### 2. 하락 상태 예시

입력 상태:
- `market_status = down`
- `price_vs_ma20_pct = -4.20`
- `ma5_vs_ma20_pct = -2.10`
- `weekly_return = -5.00`
- `volatility_level = medium`

출력 예시:
- `headline`: 하락 흐름
- `summary`: 현재 가격 흐름은 전반적으로 하락 방향으로 해석됩니다.
- `insight`: 현재 가격은 20일 평균 아래에서 움직이고 있고, 단기 평균도 중기 평균보다 낮아 최근 흐름이 비교적 약한 편입니다.
- `caution`: 최근 하락 압력이 이어질 경우 단기 변동 폭이 커질 수 있습니다.
- `beginner_note`: 하락 흐름은 최근 가격이 전반적으로 아래쪽을 향하고 있다는 뜻입니다.

---

### 3. 횡보 상태 예시

입력 상태:
- `market_status = sideways`
- `price_vs_ma20_pct = 0.2`
- `ma5_vs_ma20_pct = -0.1`
- `weekly_return = 1.1`
- `volatility_level = low`

출력 예시:
- `headline`: 횡보 구간
- `summary`: 현재 가격 흐름은 뚜렷한 방향성 없이 횡보 구간으로 해석됩니다.
- `insight`: 가격이 평균 부근에서 움직이고 있고 단기 평균과 중기 평균의 차이도 크지 않아 시장 방향성이 비교적 약한 상태입니다.
- `caution`: ""
- `beginner_note`: 횡보 구간은 가격이 크게 오르거나 내리지 않고 비슷한 범위에서 움직이는 상태입니다.

---

### 4. 주의 상태 예시

입력 상태:
- `market_status = caution`
- `daily_return = -3.8`
- `weekly_return = -8.5`
- `volatility_level = high`
- `risk_signal = true`

출력 예시:
- `headline`: 주의 필요
- `summary`: 최근 가격 변동성이 크고 낙폭도 커 주의가 필요한 구간입니다.
- `insight`: 단기 하락 폭이 큰 데다 가격 움직임도 불안정해 현재 흐름은 방향성보다 위험 관리 측면에서 먼저 살펴볼 필요가 있습니다.
- `caution`: 짧은 기간에 가격이 크게 흔들릴 수 있어 해석에 신중할 필요가 있습니다.
- `beginner_note`: 주의 상태는 가격이 갑자기 크게 움직이거나 불안정해서 조심해서 봐야 한다는 뜻입니다.

---

### 5. 데이터 부족 상태 예시

입력 상태:
- `market_status = insufficient_data`

출력 예시:
- `headline`: 데이터 부족
- `summary`: 현재 데이터만으로는 시장 상태를 안정적으로 해석하기 어렵습니다.
- `insight`: 이동평균과 변동성처럼 핵심 판단에 필요한 정보가 충분하지 않습니다.
- `caution`: 데이터가 더 쌓인 뒤 다시 해석하는 것이 적절합니다.
- `beginner_note`: 지금은 판단에 필요한 정보가 부족해서 결과를 단정하기 어렵다는 뜻입니다.

## 최종 요약

이 스킬의 핵심 역할은 다음과 같다.

- 계산된 수치와 상태 판정을 사람이 이해하기 쉬운 설명으로 바꾼다.
- 대시보드에 바로 표시할 headline, summary, insight, caution을 생성한다.
- 초보자용 쉬운 설명을 따로 제공한다.
- 과도한 단정 없이 설명 가능한 표현만 사용한다.
- 후속 UI가 바로 사용할 수 있는 구조로 결과를 반환한다.

즉, 이 스킬은 투자 대시보드 분석 파이프라인의 **최종 설명 생성 단계**이다.
